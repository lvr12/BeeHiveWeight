# Guide to contributing to the BeeHiveWeight project

Thank you for your interest in contributing to this project!

## How to contribute

1. **Report a bug**
   - Open an issue on GitHub with a clear description, steps to reproduce the problem, and the version you are using.
2. **Propose a new feature**
   - Open an “issue” indicating the desired feature, its benefits, and, if possible, some ideas for implementation.
3. **Send a pull request (PR)**
   - Fork the repository, create a branch dedicated to your change (`feature/my-feature`), make your changes, and open a pull request clearly describing what you are proposing.
   - Be sure to test your code and respect the existing structure.

## Recommendations

- Be courteous and respectful in your exchanges (see the [Code of Conduct](./CODE_OF_CONDUCT.md)).
- Keep commit messages clear and concise.
- Update the documentation if necessary.
- Check that your changes do not break existing features.
- Use explicit titles and descriptions for issues and pull requests.

## Review and merge

The project maintainers will review your contribution and may request changes before approval. Once approved, your PR will be merged into the main branch.

## Questions?

If you have a question or need help, open an issue or contact the project maintainers.
